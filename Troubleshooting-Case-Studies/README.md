# Troubleshooting Case Studies

This section contains troubleshooting case studies based on technical issues I have worked through or recreated in a lab environment.

The aim is to document the problem clearly, explain the checks carried out, and show the steps taken to reach a fix.

## Case Study Format

Each case study follows this structure:

* Issue
* Symptoms
* Initial checks
* Troubleshooting steps
* Fix
* Result
* What I learned

## Case Studies

### Windows Boot / UEFI Recovery

A case study covering a Windows boot issue where the system could not boot correctly due to boot configuration or EFI-related problems.

Skills shown:

* Windows Recovery Environment
* UEFI boot troubleshooting
* DiskPart
* EFI partition checks
* Boot repair commands

### SSD Migration and Cloning

A case study covering the process of moving a Windows installation from an older drive to an SSD.

Skills shown:

* Disk cloning
* Partition management
* Windows boot testing
* Storage troubleshooting
* Backup awareness

### Windows Update Issue

A case study covering a Windows system that had problems completing or applying updates.

Skills shown:

* Windows Update troubleshooting
* Restart testing
* System stability checks
* Basic Windows maintenance

### Disk Shrink / Partition Issue

A case study covering a situation where Windows could not shrink a partition as expected.

Skills shown:

* Disk Management
* Shadow copy awareness
* Storage troubleshooting
* Partition resizing

## Purpose

These case studies are included to show how I approach IT problems in a structured way rather than guessing.

For each issue, I try to identify the symptoms, check the basics first, use the right tools, document what changed, and confirm the result after applying a fix.
